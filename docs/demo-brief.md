# Cover Demo Brief

## What To Show First

Open the main workspace and frame it as a buildability workstation, not a dashboard:

- parcel-native review surface
- deterministic verdict with visible confidence
- citations and provenance kept close to the decision
- architect mode and sales mode on the same surface

## Demo Story Order

### 1. Parcel A: Fast-path ADU conversation

Use `Parcel A` to show:

- clean parcel confirmation
- quick verdict
- 2D and 3D envelope switch
- scenario comparison between `ADU / 1 bed` and `ADU / 3 bed`
- export-ready narrative with only one explicit caution

Talking point:

`Cover gets to a credible first answer quickly, but still makes the one open issue visible instead of hiding it.`

### 2. Parcel B: State vs. local conflict handling

Switch to `Parcel B` to show:

- hillside overlay
- interpreted conflict handling
- unresolved item count increasing
- workflow state changing to architect signoff required

Talking point:

`This is where Cover outperforms simplistic tools: it surfaces the contradiction, explains the reasoning path, and makes the human review step explicit.`

### 3. Parcel C: Portfolio / batch comparison value

Use `Parcel C` to show:

- denser-zone parcel
- comparison table in the right-side surface
- broader development upside beyond a basic ADU checker

Talking point:

`Cover is not just a single-lot widget. It can help sales and design teams compare opportunities and explain where a parcel sits in the portfolio.`

### 4. Parcel D: Premium parcel with high-stakes caution

Use `Parcel D` to show:

- coastal overlay complexity
- premium homeowner-facing story
- strong value with strong caution

Talking point:

`Cover helps the team stay ambitious without becoming reckless.`

## Audience Toggle

Use the top-right toggle to switch between:

- `Architect mode`
- `Sales mode`

Architect mode emphasizes:

- evidence
- constraint review
- scenario reading

Sales mode emphasizes:

- client-ready framing
- next-step clarity
- defensible recommendation language

## Surfaces Worth Calling Out

- `Portfolio compare`: shows multiple parcels at once
- `Constraint-aware buildable envelope`: gives 2D/3D spatial read
- `Approval workflow`: keeps deterministic handoff state visible
- `Client handoff`: translates assessment into customer-facing language without losing rigor

## Transparency Note

The current demo parcel corpus is a curated demo dataset aligned to the PRD parcel archetypes. It is designed for a polished morning demo and should later be replaced or augmented with fully verified parcel records for production evaluation.
