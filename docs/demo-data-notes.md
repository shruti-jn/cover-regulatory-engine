# Demo Data Notes

## Current State

The frontend demo uses a curated local parcel corpus in [demoStudio.ts](/Users/shruti/Software/cover-regulatory-engine/frontend/src/data/demoStudio.ts).

This corpus is designed to make the morning demo strong across four narrative types:

- baseline fast-path parcel
- state/local conflict parcel
- density / portfolio upside parcel
- premium caution parcel

## Why This Exists

The PRD defines strong parcel archetypes and rich verification expectations, but the locked ZIMAS-verified parcel set still contains placeholder values in the product definition.

For demo readiness, we need:

- multiple believable parcels
- visible scenario variation
- portfolio comparison
- deterministic customer-facing storytelling

## What This Corpus Is

- a demo-focused parcel set
- aligned to the PRD archetypes
- suitable for product walkthroughs and UI validation

## What This Corpus Is Not

- not a claim of official parcel verification
- not a production regulatory source of truth
- not a substitute for the later verified parcel set

## Next Production Step

Replace or augment the demo corpus with a verified parcel library sourced from the official verification workflow described in the PRD and linked evidence sources.
